# Aereth Blockout Standards

## Purpose

A blockout is a playable prototype of space. It is not final art.

## Rules

1. Readability before detail.
2. Pathing before decoration.
3. Landmarks before micro-builds.
4. One strong visual idea per area.
5. Avoid giant areas that cannot be filled with gameplay.
6. Capture coordinates immediately after blockout.
7. Test as non-OP whenever possible.

## MMO Starter Area Standards

A starter area should answer:

- Where am I safe?
- Where do I go next?
- What is strange about this world?
- What is the first problem I can help solve?
- What larger world is being implied?

## Aereth Visual Rules

- Blue-teal glow = memory/anomaly/Fragment pressure. Use sparingly.
- Gold/warm light = human/settled recovery and lantern safety.
- Purple/black = fracture/void danger.
- Broken roads = continuity damaged but not erased.
- Archives/ledgers = survival through memory and record.

## Scope Rule

If a builder says "while we are here, we could also build the entire coastline", gently confiscate their keyboard.
