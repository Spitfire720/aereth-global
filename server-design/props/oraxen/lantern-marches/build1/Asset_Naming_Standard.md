# Oraxen Asset Naming Standard - Aereth Build 01

## Item IDs

Use prefix:

```text
lm_
```

Example:

```text
lm_registry_ledger
```

## Model paths

Use:

```text
aereth/lantern_marches/<asset_name>
```

Example:

```text
aereth/lantern_marches/registry_ledger
```

## Texture paths

Use:

```text
aereth/lantern_marches/<asset_name>.png
```

## Naming rules

- lowercase only
- underscores only
- no spaces
- no version numbers in live item IDs
- avoid plugin names in IDs
- use canon object names where possible

## Future prefixes

```text
lm_ = Lantern Marches
lr_ = Lantern's Rest
br_ = Bent Road
hf_ = Hollowglass / Hollowfen visuals, if separated later
```
