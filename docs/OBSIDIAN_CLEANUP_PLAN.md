# Obsidian Cleanup Plan

## Purpose

Obsidian should become the lore graph and thinking layer for Aereth.

It should not replace Notion command docs or GitHub source docs.

## Source of truth split

| Material | Destination |
| --- | --- |
| Current decisions | Notion |
| Versioned code/docs | GitHub |
| Lore graph / linked thinking | Obsidian |
| Binaries and backups | Drive / server storage |

## Cleanup order

1. Finish repo cleanup.
2. Identify Obsidian vault root.
3. Create top-level folders.
4. Move scattered lore notes into folders.
5. Add index notes.
6. Link core concepts.
7. Mark stale/generated lore as archive.
8. Sync public-facing lore back into Notion World Bible and website wiki.

## Rule

Obsidian is for connected lore thinking.

Notion is for current project control.

GitHub is for versioned implementation.
